# 全國廁所快找 - 系統分析與技術規格報告 (Final Edition)

## 1. 系統概述
本系統為一款專為行動裝置（iPhone/Android）設計的 Progressive Web App (PWA)，核心目標是提供高辨識度、極速響應的公廁查詢服務。系統採 **Serverless 靜態架構**，所有地理運算與資料過濾均在前端完成。

## 2. 軟體架構 (Software Architecture)
- **前端核心**: 原生 HTML5 / CSS3 / JavaScript (ES6+)。
- **地圖引擎**: Leaflet.js 1.9.4。
- **資料儲存**: 靜態 JSON 封裝 (`toilets.json`)，目前包含全台約 2.5 萬筆地點。
- **離線技術**: Service Worker (Cache Storage API)，支援 100% 離線地圖與資料存取。
- **自動化維護**: Python ETL 腳本，負責 API 同步、資料清洗與版本更新。

## 3. 資料處理規格 (Data Processing & ETL)
### 3.1 原始資料來源
- **來源名稱**: 環境部環境資料開放平臺 - 全國公廁建檔資料 (FAC_P_07)。
- **原始筆數**: 約 4.6 萬筆（按男/女/無障礙單元拆分）。

### 3.2 聚合演算法 (Aggregation Logic)
為避免地圖上標記重疊導致操作困難，系統實作了「站點聚合」：
- **Key 產生**: `hash(緯度.5f, 經度.5f, 名稱.prefix(5))`。
- **處理內容**: 
    - 將相同座標地點的「男、女、無障礙、性別友善」標記合併至單一站點。
    - `has_diaper` 欄位採 `OR` 運算（任一單元有尿布檯即視為該站點具備）。

## 4. 核心功能規格 (Core Functionality)
### 4.1 定位狀態機 (Location State Machine)
系統嚴格區分定位基準點，分為以下三種狀態：
1. **自動 GPS 模式 (預設)**: 
    - 標記: 🔴 紅色大頭針。
    - 邏輯: 地圖中心隨 `navigator.geolocation` 實時更新。
2. **手動選點模式**: 
    - 觸發: 點擊「手動定位」按鈕。
    - 邏輯: 移除紅點，暫停位移跟隨，等待地圖點擊事件。
3. **手動鎖定模式**: 
    - 觸發: 地圖點擊完成。
    - 標記: 🔵 藍色標記 (📍)。
    - 邏輯: 鎖定座標，防止手指滑動造成誤觸移位。

### 4.2 搜尋與路徑規劃
- **最近搜尋**: 採 Haversine 公式計算直線距離，找出最小值的公廁站點。
- **視覺導航線**: 
    - **最近連線 (綠色)**: 具備持久性，僅在定位點變更時清除。
    - **選取連線 (橙色)**: 隨面板開啟/切換點選目標而動態更新。
- **視覺規範**: 雙層實線發光效果（2px 核心 + 6px 光暈），確保強光下可見。

## 5. 視覺與操作設計 (UI/UX Standards)
- **配色策略**: 深黃背景 (#fbc531) 與純黑文字，提供極限對比。
- **圖標規範**: 深藍圓角矩形路牌風格，內嵌白色 WC 圖示。
- **動態渲染 (Dynamic Density)**: 
    - Zoom 18+: 上限 1000 點（全量顯示）。
    - Zoom 16: 上限 600 點。
    - Zoom 14: 上限 300 點。
- **資料互動**: 所有 UI 顯示之地址與設施文字均可長按複製，利於轉發或偵錯。

## 6. 維護與更新機制
透過 `build_app.py` 實現一鍵發布：
1. 抓取 API 資料。
2. 執行 `clean_data()` 聚合。
3. 同步至根目錄 `toilets.json`。
4. **自動更新版本號**: 產生帶時間戳的 `CACHE_NAME` 寫入 `sw.js`，驅動 PWA 自動熱更新。
